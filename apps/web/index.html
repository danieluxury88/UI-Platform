Calendar milestone

Month-first planning is now wired into the shared system.

The demo uses a controlled anchor date, shared navigation events, and compact event chips inside the month grid.

Controlled state

Selection and activation stay separate.

Selected date

Pick a date or activate an event chip to inspect the current state.

The calendar starts on Monday for this demo and keeps the month view as the only supported view for now.

Day view

The same shell now supports a narrow agenda-style day view.

This slice keeps the API controlled and reuses the same shared event model as the month view.

Day focus

One day, compact events, and a single selection target.

Selected day

Choose the visible day or activate an event chip to inspect the focused agenda.

The day view stays thin on purpose so week and year can build on the same shared shell later.

Week view

The shell now covers a narrow seven-day planning slice.

This keeps the week contract controlled without adding collisions, drag and drop, or a multi-week bridge.

Week focus

Seven days, compact events, and the same event model.

Selected day

Move the week anchor, select a day, or activate a chip to inspect the visible range.

The week view is intentionally thin so later week-specific layout can evolve without changing the shell contract.
